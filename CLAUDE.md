# CLAUDE.md

Claude Code（claude.ai/code）がこのリポジトリで作業する際のガイドです。

## クイックスタート

### セットアップ
1. Python 3.9以上がインストールされていることを確認
2. 依存パッケージをインストール：`pip install -r requirements.txt`
3. [Google AI Studio](https://aistudio.google.com/app/apikey) からGemini APIキーを取得
4. `.env.example` をコピーして `.env` を作成し、APIキーを追加：`GEMINI_API_KEY=your_key_here`

### アプリの起動
```bash
streamlit run app.py
```
ブラウザで `http://localhost:8501` が開き、左側のサイドバーに3つの機能が表示されます。

### よく使うコマンド
```bash
# アプリを起動（ファイル変更時に自動リロード）
streamlit run app.py

# カスタムポート指定で起動
streamlit run app.py --server.port 8502

# Streamlitのキャッシュをクリアしてリセット
streamlit cache clear

# 開発前にAPI接続をテスト
python test_api.py
```

### テストとデバッグ
```bash
# API接続と基本機能をテスト
python test_api.py
```
このスクリプトはAPIキー、トークン推定、入力長チェックを検証します。開発前に環境が正しくセットアップされているか確認に使用します。

開発中に単一の機能をテストする場合：
1. `streamlit run app.py` でアプリを起動
2. 左側のサイドバーから機能を選択
3. テスト入力を送信して、出力が正しいか確認
4. ブラウザのコンソールとターミナルでエラーをチェック

**注意：** Streamlitはファイル変更時にスクリプト全体を再実行します。開発中は `SYSTEM_PROMPT` 定数を変更して自動リロードをトリガーします。

### 依存パッケージ
- **streamlit** (>=1.36.0) — Webインターフェース構築用UIフレームワーク
- **google-genai** (>=0.3.0) — Gemini API SDK
- **python-dotenv** (>=1.0.1) — `.env` ファイルから環境変数を読み込み

## アーキテクチャ

このプロジェクトはGoogle Gemini APIを使用した3つのAIライティングツールを提供するStreamlit MVPです。

### コア設計パターン
- **モジュール化された機能**: 各ライティングツール（`blog_writer.py`、`email_reply.py`、`summarizer.py`）は独立したモジュールで、それぞれが `render()` 関数を持ち、独自のStreamlit UIとプロンプトロジックを管理
  - 利点：新機能追加時に既存機能に影響を与えず実装可能
- **一元化されたAPI層**: すべてのGemini API呼び出しは `utils/gemini_client.py` を通す
  - 利点：APIの仕様変更があった場合、このファイルだけ編集すれば全機能に反映される
- **システムプロンプト**: 各機能が独自の `SYSTEM_PROMPT` を定義してAIの動作を制御
  - 例：ブログライターは「プロのブログ執筆者」、メール返信は「ビジネスアシスタント」という役割設定
- **設定管理**: `config.py` がAPIキーと使用モデル（`gemini-2.5-flash`）を一元管理
  - 異なるモデルを試したい場合はこのファイルを編集するだけ

### 新機能の追加方法
1. `features/` に新しいファイルを作成（例：`features/new_tool.py`）
2. AIの役割を説明する `SYSTEM_PROMPT` 定数を定義
3. Streamlitウィジェットでユーザー入力を収集する `render()` 関数を実装
4. `generate_text(prompt, system_instruction=SYSTEM_PROMPT, temperature=...)` でコンテンツ生成
5. `app.py` の `MENU` 辞書に登録

**例：** 「句読点チェッカー」を追加したい場合、`features/punctuation_checker.py` を作成して `app.py` の `MENU` に登録するだけです。

### モデル動作のカスタマイズ
- **Temperature（温度）**: 出力のランダムさを制御（0.3で事実的、0.8で創造的）
  - 要約タスク：0.3（事実性重視）
  - ブログ執筆：0.8（創造性重視）
  - メール返信：0.6（バランス型）
- **モデル選択**: `config.py` の `DEFAULT_MODEL` を変更、または `generate_text()` に `model=` パラメータを指定
- **システムプロンプト**: 各機能の `SYSTEM_PROMPT` を編集してAIのトーンやスタイルを調整

## ファイル構成

```
app.py                 # Streamlitエントリーポイント、サイドバーナビゲーション
config.py              # APIキーとモデル設定

  gemini_client.py     # Gemini APIラッパー（generate_text()関数）
features/
  blog_writer.py       # ブログ記事生成
  email_reply.py       # メール返信生成
  summarizer.py        # テキスト要約
requirements.txt       # 依存パッケージ
.env                   # APIキー（バージョン管理外）
README.md              # ユーザー向けドキュメント
```

## 開発ノート

### 実装詳細
- **テストなし**: このMVPにはユニットテストや統合テストが含まれていません
- **エラーハンドリング**: 各機能がAPI呼び出しの例外をキャッチし、`st.error()` でユーザーフレンドリーなエラーメッセージを表示
- **ダウンロード機能**: ブログライターのみMarkdown形式でエクスポート可能。他のツールはテキストエリアに出力
- **Streamlit状態管理**
  - ウィジェット状態は実行間でリセット
  - フォーム送信をトリガーに計算を実行する設計
  - `st.session_state` はスクリプト再実行時も値が保持される
  - API呼び出し結果などをキャッシュして、フォーム再送信時に再利用可能
- **Temperature調整**
  - `blog_writer` (0.8)：創造的なコンテンツ生成
  - `email_reply` (0.6)：専門的だがバランスの取れたトーン
  - `summarizer` (0.3)：事実性を重視した圧縮テキスト

### Streamlit開発ワークフロー
- **ホットリロード**: Streamlitが `.py` ファイルを監視し、保存時に再実行
  - `SYSTEM_PROMPT` や定数を変更して保存するだけでブラウザに反映
- **ローカルでの変更テスト**: プロンプトやパラメータを試しながら調整可能
  - 例：`blog_writer.py` の `SYSTEM_PROMPT` を編集して、トーン変更をすぐテストできる
- **デバッグ出力**
  - `st.write()` ：アプリ画面に表示
  - `print()` ：ターミナル/コンソールに表示
- **パフォーマンス**: API呼び出しなど時間がかかる処理は UI をブロック
  - `with st.spinner('処理中...')` を使用してユーザーに待機状態を通知
- **状態検査**: ブラウザ右上メニューのStreamlitデバッグからセッション状態を確認可能
  - キャッシュ値のデバッグに便利

### 既知の制限と制約
- **APIレート制限**: Gemini無料枠は1分あたり約15リクエスト
  - この制限に達するとエラーが表示される
  - 開発時は短いテキストで試すか、複数のAPIキーを用意
- **コンテキスト長制限**: 約30,000トークン以上のテキストを送信すると失敗する可能性
  - 要約とメール返信機能で問題になりやすい
- **キャッシング機能なし**: 同じプロンプトでも毎回API呼び出しされるため配額を無駄に消費
  - 複数回テストする際は `st.session_state` を活用してキャッシュ
- **入力長チェックなし**: ユーザーが大きなテキストを貼り付けるとエラーになる可能性
  - ユーザーへの事前警告がない
- **トークン数計算機能なし**: リクエスト前にトークン使用量の警告ができない

### 今後の改善（優先度順）
1. **入力検証機能**: API送信前にテキスト長をチェックし、制限に近い場合は警告
2. **応答キャッシング**: `st.session_state` にプロンプトと出力を保存して重複API呼び出しを回避
3. **トークン推定**: 生成前に推定トークン数と費用警告を表示
4. **API エラー詳細化**: 認証失敗、レート制限、配額超過を区別したエラーメッセージ
5. **マルチモデル対応**: GeminiやClaudeなど複数モデルをドロップダウンで切り替え可能に
6. **出力履歴**: セッション中の生成結果を保存して、ユーザーが以前の結果を確認・編集可能に
7. **バッチ操作**: 1度に複数のバリエーション生成（例：ブログアウトライン3パターン）

### デバッグとトラブルシューティング

#### よくあるエラー
- **APIキーが見つからない**: `.env` ファイルが存在し、`GEMINI_API_KEY=` 行があるか確認
  - `config.py` がキーなしの場合 `RuntimeError` を発生させる

- **"400 Invalid Argument" エラー**: システムプロンプトが空またはプロンプトテキストに無効な文字
  - 各機能の `SYSTEM_PROMPT` 定義を確認

- **"429 Resource exhausted"（レート制限）**: 無料枠は1分あたり約15リクエスト
  - 60秒待つ、または異なるAPIキーを使用
  - 開発時は短いテキストで試すか複数キーを用意

- **"403 Permission denied"**: APIキーが無効または失効
  - [Google AI Studio](https://aistudio.google.com/app/apikey) で再生成

- **生成が遅い**: 無料枠がスロットル中の可能性
  - 有料枠を使用するか、より軽量なモデル（`gemini-1.5-flash`）を試す

- **出力が途中で途切れている**: API側がトークン制限に達した可能性
  - 入力長を減らすか、より短い出力をリクエスト

- **Streamlit "Script changed" 警告**: Pythonファイル変更時の正常な動作
  - Streamlitが自動的にスクリプト全体を再実行する

- **サイドバーに機能が表示されない**: 機能モジュールが `app.py` の `MENU` 辞書に登録されているか確認
  - `render()` 関数が実装されているか確認
